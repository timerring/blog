The Logging Best Practice

March 7, 2025 · 1 min read · Page View:
Tutorial
Python
If you have any questions, feel free to comment below.

The logging module is one of the most basic and most used modules in Python. It is used to record messages and errors.

Logging level #

The logging module provides several levels of logging, each with a different purpose and severity. The main levels are:

  • DEBUG: Detailed information for debugging purposes.
  • INFO: General information about the program’s operation.(Most used)
  • WARNING: An indication that something unexpected happened but the program is still running.
  • ERROR: An error occurred that prevented the program from performing a specific function.
  • CRITICAL: A serious error that prevents the program from continuing execution.

Logging handler #

The logging module provides several handlers, each with a different purpose and severity. The main handlers are:

  • StreamHandler: Outputs log messages to the console.
  • FileHandler: Outputs log messages to a file.
  • RotatingFileHandler: Outputs log messages to a file and rotates the file when it reaches a certain size.
  • TimedRotatingFileHandler: Outputs log messages to a file and rotates the file at a certain interval.

Some tips #

  1. I highly recommend logging the message based on time rather than on a basis of once execution. Because if there are no errors for a long time, the logs will accumulate to a large quantity, making them unreadable or even difficult to open.
